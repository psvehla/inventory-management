Factory Inventory Management System

System Architecture & Technical Overview

Tech Stack

Frontend

Vue 3 + Vite

Composition API, Vue Router, Axios, scoped CSS, i18n (EN/JA)

Backend

Python FastAPI

Pydantic models, Uvicorn, CORS middleware, query-param filtering

Data Layer

In-Memory JSON

7 JSON files loaded at startup via mock_data.py. No database.

Tooling

Dev Environment

Vite dev server (:3000), Uvicorn (:8001), uv package manager, pytest

System Architecture

Data Flow

1

User Action

Selects filter or navigates to a view

→
2

Filter State

useFilters() composable updates reactive refs

→
3

API Call

api.js sends GET with query params via Axios

→
4

Server Filter

FastAPI applies warehouse, category, status, month filters

→
5

Validate

Pydantic models serialize response

→
6

Render

Vue computed properties update, template re-renders

Frontend Views

Dashboard

/

KPI cards with progress bars, order health donut chart, inventory value by category, monthly revenue trend, warehouse distribution

Inventory

/inventory

Searchable table of stock items with SKU, quantity, reorder points, status badges (healthy/low/critical)

Orders

/orders

Order table with status tracking (delivered, shipped, processing, backordered), date and value columns

Demand Forecast

/demand

Trend analysis cards (increasing/stable/decreasing), current vs forecasted demand, category breakdown

Spending Analytics

/spending

Cost breakdown by category (procurement, operational, labor, overhead), monthly trends, transaction history

Reports

/reports

Quarterly performance reports, monthly trends, revenue analysis, fulfillment metrics

  • main.jsVue app entry, router config (6 routes)

Backend (server/)

  • main.pyFastAPI app, endpoints, Pydantic models, filter logic
